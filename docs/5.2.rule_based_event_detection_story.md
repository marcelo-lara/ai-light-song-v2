# EPIC 5 - Story 5.2
## Rule-Based Detection with Adaptive Thresholds

## Goal

Detect obvious transition and sustained-state events with deterministic rules before adding higher-level classifiers or ML models.

## Inputs

- event vocabulary and schema from Story 5.1
- normalized event feature layer from Story 4.4
- section windows from Story 2.5
- optional genre profile guidance from Story 6.1

## Dependencies

- deterministic threshold configuration
- explicit rule evidence payloads for each accepted event
- stable fallback behavior when multiple event types compete for the same window

## Processing Steps

1. Evaluate transition-event rules for `build`, `breakdown`, `drop`, `fake_drop`, `impact_hit`, and `pause_break`.
2. **Adaptive Moving-Average Thresholding**:
   - Replace fixed energy thresholds with an **Adaptive Thresholding** strategy.
   - Triggers for events like `impact_hit` or `drop` should only occur when energy in a specific band exceeds its local moving average (LMA) by a configurable ratio (e.g., 1.5x). This ensures the detector captures transients relative to the current musical context rather than absolute volume.
3. Evaluate sustained-state rules for `groove_loop`, `atmospheric_plateau`, `tension_hold`, and `no_drop_plateau`.
4. Attach rule evidence, threshold values, source windows, and nearby section references to each accepted candidate.
5. Emit configurable threshold presets so later validation can tune the rules per genre without changing the output schema.
6. **Deterministic Randomization**: Use the Mersenne Twister (MT19937) for any non-deterministic steps in candidate selection to ensure reproducible debug sessions.

## Output Artifacts

- `data/artifacts/<Song - Artist>/event_inference/rule_candidates.json`

## Acceptance Criteria

- Clear benchmark cases for transition and sustained-state events can be detected without ML.
- Thresholds are configurable and deterministic.
- Every emitted candidate includes explicit rule evidence rather than only a label and score.
- Sustained-state events can span multi-second windows and coexist with section labels.
- **Adaptive Logic**: Triggers correctly adapt to the local energy context, detecting hits in quiet sections as effectively as in loud sections by using relative energy ratios.

## Validation

- Compare rule candidates against benchmark songs with known drops, breakdowns, pauses, or held states.
- Confirm repeated runs on the same song produce identical candidate windows and evidence payloads.
- Confirm low-confidence or ambiguous windows are preserved for review instead of being silently discarded.

## Failure Cases

- Rules only detect isolated spikes and miss obvious multi-second states.
- Thresholds are embedded in code paths without a documented configuration surface.
- Candidate events cannot explain why they fired.

## Out of Scope

- Drop subtyping beyond a generic baseline label
- Human override persistence
- ML-based sequence labeling
