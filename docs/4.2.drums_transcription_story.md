# EPIC 4 - Story 4.2

## Drums Transcription

## Goal

Translate the drums stem into a compact event stream of kick, snare, and hat hits that can be reviewed quickly and reused by later rhythmic logic.

In phase 1, this story produces a simple producer-scoped review/debug artifact rather than expanding the canonical `layer_b_symbolic.json` contract.

## Inputs

- Drums stem from Story 1.1
- Source song mix from `data/songs/<Song - Artist>.mp3`
- Canonical beat grid from Story 1.2
- Raw drums note cache from Story 4.1 when auxiliary model output is useful during review

## Dependencies

- Omnizart drum transcription from the `audiohacking/omnizart` fork
- A usable Omnizart drum checkpoint directory
- No full-kit taxonomy beyond `kick`, `snare`, and `hat` in phase 1 review output

## Phase 1 Default Implementation

- Prefer the isolated drums stem as the primary source.
- Run Omnizart drum transcription on the isolated drums stem and preserve the raw MIDI cache it produces.
- Parse the Omnizart drum MIDI into the review artifact by collapsing supported GM drum pitches into `kick`, `snare`, and `hat`.
- Keep unsupported or unmapped drum-note pitches explicit as `unresolved` instead of coercing them into a supported label.
- Align accepted events to the canonical beat grid when the onset falls within the allowed timing tolerance.
- Preserve explicit debug source references for both the full mix and the isolated drums stem in the artifact metadata.
- Fail explicitly when Omnizart is unavailable or when a usable checkpoint directory is not configured. Do not fall back to a custom heuristic detector.
- Emit a single ordered JSON artifact that is easy to inspect during review and debugging.

## Processing Steps

1. Load the drums stem and canonical beat grid.
2. Run Omnizart drum transcription on the drums stem.
3. Preserve the raw Omnizart MIDI output under `data/artifacts/<Song - Artist>/symbolic_transcription/omnizart/drums.mid`.
4. Record debug source references for `data/songs/<Song - Artist>.mp3` and `data/stems/<Song - Artist>/drums.wav` in the artifact metadata.
5. Map Omnizart drum-note pitches into `kick`, `snare`, `hat`, or unresolved review rows.
6. Attach aligned bar and beat fields when a valid beat match exists.
7. Summarize event counts and obvious quality concerns for quick inspection.
8. Write the producer-scoped review artifact to `data/artifacts/<Song - Artist>/symbolic_transcription/drum_events.json`.

## Output Artifacts

- `data/artifacts/<Song - Artist>/symbolic_transcription/drum_events.json`
- `data/artifacts/<Song - Artist>/symbolic_transcription/omnizart/drums.mid`

## Acceptance Criteria

- The artifact represents the main kick, snare, and hat activity with labels that are audibly plausible.
- The output is easy to inspect manually because rows are ordered by time and every event keeps an explicit type and confidence.
- Low-confidence or unresolved hits remain visible for debugging instead of being silently converted into a supported class.
- The raw Omnizart MIDI cache is preserved for inspection alongside the normalized JSON artifact.
- The artifact metadata preserves explicit debug source references for both the source mix and the drums stem without copying those audio files into the artifact tree.
- The phase-1 artifact remains producer-scoped and does not silently change the canonical `layer_b_symbolic.json` schema.
- The implementation fails explicitly when Omnizart or its checkpoints are unavailable instead of substituting a heuristic detector.
- Running the story on `What a Feeling - Courtney Storm.mp3` yields a recognizable backbeat and hat pulse that can be reviewed without opening raw audio tooling first.

## Validation

- Run the story against `What a Feeling - Courtney Storm.mp3`.
- Confirm that obvious downbeat kicks and backbeat snares land near the audible drum accents.
- Confirm that hat events follow the audible high-frequency pulse without exploding into implausible density.
- Confirm the raw Omnizart MIDI cache exists and contains only the supported three-instrument drum projection when the default Omnizart drum model is used.
- Inspect summary counts and quality flags before promoting the approach to broader batch usage.

## Failure Cases

- Omnizart checkpoints are missing or incompatible with the runtime.
- Omnizart emits only sparse kick/snare activity and the artifact fails to expose a recognizable backbeat.
- Unsupported drum-note pitches appear in the raw MIDI and are hidden instead of preserved as unresolved review rows.
- The review artifact hides uncertainty by dropping unresolved hits completely.
- The implementation copies large debug-audio files into the artifact tree instead of preserving explicit source-path references.

## Out of Scope

- Full-kit drum taxonomy beyond kick, snare, and hat
- Ghost-note transcription
- Human-edited drum cleanup

## Note

- The required implementation source for phase 1 is the `audiohacking/omnizart` fork: https://github.com/audiohacking/omnizart
- Do not substitute a custom onset-detector or classifier fallback when Omnizart is missing or unusable.
