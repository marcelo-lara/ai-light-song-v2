# EPIC 8 - Story 8.2

## Master Sync and Waveform Anchor

## Goal

Establish the internal debugger playback surface and the shared timeline source of truth for artifact inspection.

This story is for the internal artifact debugger under `/ui/`. It is not a production consumer UI story.

## Inputs

- `data/songs/<Song - Artist>.mp3`
- `data/output/<Song - Artist>/info.json`
- `data/artifacts/<Song - Artist>/essentia/beats.json`

## Processing Steps

1. Load song metadata and timing context for a selected song directory.
2. Initialize the debugger playback shell from the `/ui` application and preload browser audio metadata for the selected song.
3. Treat the browser-local shared cursor as the authoritative paused-state position, and sync it from audio playback time while playback is active.
4. Expose playhead updates to downstream debugger lanes and overlays.
5. Serve mounted song files with byte-range support so the browser can seek paused audio and resume playback from the current cursor.

## Output

- browser-only playback state inside the debugger

No generated files are written to `data/artifacts/` or `data/output/`.

## Acceptance Criteria

- The debugger can load a song and display core metadata.
- The playback surface runs in the separate Compose `ui` service.
- Selecting a song preloads enough browser audio metadata for cursor-driven seeking and play-from-cursor transport behavior.
- The `/data/songs/<Song>.mp3` path served by the debugger supports HTTP byte-range requests required for browser seeking.
- Pressing `Play` resumes from the current shared cursor position rather than resetting to the start of the song, even when the browser has not yet loaded song metadata for the audio element.
- Playback state is read-only against generated data.
- No debugger-generated files are written into generated-data folders.
