# 💡 EPIC 5 — Story 5.1 Energy → Lighting Mapping

## 🎯 Goal  
Translate analyzed audio energy and musical features into **DMX lighting behaviors**.

This is the layer where **analysis becomes action**.

---

## 🛠️ Tools

- Internal rule engine (Python)
- DMX abstraction layer (Art-Net / sACN)
- Optional:
  - LLM (for high-level interpretation)
  - Preset library (lighting patterns)

---

## ⚙️ Tasks

### 1. Input Aggregation

Consume outputs from:

- EPIC 3 → symbolic features
- EPIC 4.1 → energy features
- EPIC 4.2 → sections

Key inputs:

- loudness
- spectral flux
- onset strength
- note density
- bass movement
- section type

---

### 2. Normalize Feature Space

Convert all inputs into normalized ranges:

- intensity → 0–1
- motion → 0–1
- density → 0–1

---

### 3. Define Lighting Dimensions

Map features into lighting controls:

| Feature | Lighting Parameter |
|--------|------------------|
| loudness | dimmer intensity |
| spectral centroid | color temperature / hue |
| spectral flux | movement speed |
| onset strength | strobe / flash |
| note density | effect complexity |
| bass movement | proton lights / pulse |
| section type | scene selection |

---

### 4. Rule-Based Mapping Engine

Implement mapping logic:

#### Intensity

```
dimmer = loudness * master_intensity
```

#### Movement Speed

```
movement_speed = spectral_flux * max_speed
```

#### Strobe Activation

```
if onset_strength > threshold:
    strobe = True
```

#### Color Mapping

```
if centroid < 1500:
    color = warm
else:
    color = cool
```

---

### 5. Section-Based Overrides

Adjust behavior based on section:

#### Intro
- low intensity
- slow movement

#### Verse
- moderate intensity
- rhythmic motion

#### Chorus
- high intensity
- fast movement
- strobes enabled

#### Bridge
- reduced intensity
- contrast effects

---

### 6. Generate Lighting Events

Produce time-based events:

```json
{
  "time": 60.0,
  "duration": 4.0,
  "fixtures": {
    "par_1": {
      "dimmer": 0.9,
      "color": "blue"
    },
    "moving_head": {
      "pan": 120,
      "tilt": 45,
      "speed": 0.8
    }
  }
}
```

---

### 7. Smoothing & Transitions

- Apply interpolation between frames
- Avoid abrupt jumps
- Respect fixture limits

---

## 📦 Output Schema

```json
{
  "lighting_events": [
    {
      "time": 0.0,
      "duration": 1.0,
      "intensity": 0.5,
      "color": "warm",
      "movement_speed": 0.2
    }
  ]
}
```

---

## 🧪 Acceptance Criteria

- Lighting reacts in sync with music
- Chorus sections visibly more intense than verses
- No abrupt flickering (unless intentional)
- Movement feels musical, not random

---

## ⚠️ Common Pitfalls

- Over-reactivity → chaotic lights
- Ignoring sections → flat show
- No smoothing → jittery output

---

## 🚀 Future Enhancements

- Style presets (EDM, rock, ambient)
- Fixture-aware mapping (moving heads vs pars)
