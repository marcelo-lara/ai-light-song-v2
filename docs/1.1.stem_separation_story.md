# EPIC 1 - Story 1.1

## Stem Separation

## Goal

Split the source song into reusable stems for downstream timing, harmonic, symbolic, and energy analysis.

## Inputs

- Source song from `data/songs/<Song - Artist>.mp3`

## Dependencies

- Docker runtime with NVIDIA GPU access
- Demucs or equivalent stem-separation model
- `ffmpeg` for format conversion and normalization checks

## Processing Steps

1. Validate the source song path and audio readability.
2. Run stem separation to extract at minimum: `bass`, `drums`, `harmonic`, `vocals`.
3. Convert stems to analysis-friendly `.wav` format when the separation tool emits another format.
4. Normalize stems to a consistent peak or loudness target suitable for downstream analyzers.
5. Persist stems to `data/stems/<Song - Artist>/`.
6. Record the output stem paths and generation metadata for reuse by later stories.

## Output Artifacts

- `data/stems/<Song - Artist>/bass.wav`
- `data/stems/<Song - Artist>/drums.wav`
- `data/stems/<Song - Artist>/harmonic.wav`
- `data/stems/<Song - Artist>/vocals.wav`
- optional metadata file describing generator version and normalization settings

## Output Schema

```json
{
  "schema_version": "1.0",
  "song_name": "Song - Artist",
  "generated_from": {
    "source_song_path": "data/songs/Song - Artist.mp3",
    "engine": "demucs"
  },
  "stems": {
    "bass": "data/stems/Song - Artist/bass.wav",
    "drums": "data/stems/Song - Artist/drums.wav",
    "harmonic": "data/stems/Song - Artist/harmonic.wav",
    "vocals": "data/stems/Song - Artist/vocals.wav"
  }
}
```

## Acceptance Criteria

- All four required stems are generated.
- Stems are readable by downstream analyzers.
- Output naming is stable and deterministic.
- The process can be rerun without corrupting cached outputs.

## Validation

- Confirm each stem file exists and is non-empty.
- Confirm sample rate and channel format are supported by downstream tooling.
- Confirm the harmonic and bass stems are usable inputs for EPIC 2 and EPIC 3.

## Failure Cases

- Missing source song file
- Separation model failure
- Corrupt or zero-length output stem
- Unsupported audio codec or sample format

## Out of Scope

- Artistic remixing of stems
- Manual stem cleanup
- Multi-model ensemble separation
