# EPIC 3 - Story 3.4

## LLM-Friendly Abstraction

## Goal

Convert symbolic features into concise, musician-readable descriptors that are useful to both humans and downstream prompt-based workflows.

## Inputs

- Symbolic summaries from Story 3.2
- Aligned phrase and section information from Story 3.3

## Dependencies

- Deterministic descriptor templates
- Optional later LLM refinement, but not required for baseline generation

## Processing Steps

1. Convert core symbolic metrics into a controlled vocabulary.
2. Build short global and phrase-level descriptions using deterministic templates.
3. Avoid vague language that hides the actual feature values.
4. Persist both structured metrics and human-readable descriptions.

## Output Schema

```json
{
  "description": "Repeated mid-register pattern with rising contour, moderate density, and leaping bass movement."
}
```

## Acceptance Criteria

- The description is understandable by a musician.
- The wording reflects the underlying metrics rather than generic prose.
- Repeated runs on the same artifact yield the same text unless upstream features change.

## Validation

- Compare the text against the structured symbolic summary.
- Confirm the description does not claim features missing from the actual metrics.

## Failure Cases

- Overly generic text that fits every song
- Descriptions contradict the structured symbolic data
- Abstraction layer becomes dependent on an external LLM to function

## Out of Scope

- Creative lyric interpretation
- Genre storytelling beyond the symbolic descriptor set
