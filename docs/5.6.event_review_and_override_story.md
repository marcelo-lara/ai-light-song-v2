# EPIC 5 - Story 5.6

## Confidence, Review, and Override Workflow

## Goal

Expose event confidence, ambiguity, and editable review outputs so humans can confirm, correct, or replace machine-detected events without modifying raw analysis artifacts.

## Inputs

- event vocabulary and schema from Story 5.1
- machine-detected events from Story 5.5
- optional identifier hints from Story 5.4

## Dependencies

- stable confidence band definitions
- explicit candidate ranking for ambiguous windows
- separate storage for machine output and human-authored overrides

## Processing Steps

1. Define confidence bands such as high, medium, and low, and document their numeric thresholds.
2. Flag ambiguous windows where multiple event types or subtypes score similarly.
3. Export review-friendly machine output in JSON or markdown without requiring edits to raw analysis artifacts.
4. Support confirm, delete, retime, relabel, and note-taking operations in a separate override file.
5. Preserve provenance so the final export can distinguish machine-only events from human-confirmed or human-edited events.

## Output Artifacts

- `data/output/<Song - Artist>/song_events.review.json`
- `data/output/<Song - Artist>/song_events.review.md`
- `data/output/<Song - Artist>/song_events.overrides.json`

## Acceptance Criteria

- Ambiguous detections are visible instead of being silently forced into one label.
- Reviewers can confirm, delete, retime, relabel, and annotate events without editing raw analysis files.
- Machine and human-authored event versions remain distinguishable and reproducible.

## Validation

- Confirm override files can be applied deterministically to the machine output.
- Confirm low-confidence windows preserve ranked alternatives and scores.
- Confirm the final merged event list records whether each event was machine-only, human-confirmed, or human-edited.

## Failure Cases

- Review output loses the original machine evidence or timing.
- Human edits overwrite machine output in place with no provenance.
- Low-confidence windows omit the candidate labels that a reviewer needs.

## Out of Scope

- Interactive GUI design
- ML model training
- Fixture-aware lighting orchestration
