# EPIC 5 - Story 5.2

## Event Feature Normalization and Timeline Alignment

## Goal

Normalize and align the upstream audio, symbolic, harmonic, and section layers into an event-inference feature set that can be queried on a shared timeline.

## Inputs

- `data/artifacts/<Song - Artist>/layer_a_harmonic.json`
- `data/artifacts/<Song - Artist>/layer_b_symbolic.json`
- `data/artifacts/<Song - Artist>/layer_c_energy.json`
- `data/artifacts/<Song - Artist>/section_segmentation/sections.json`
- canonical beat and bar timing from Story 1.2
- optional producer-scoped genre guidance from Story 2.5

## Dependencies

- stable upstream layer schemas
- deterministic alignment between raw timestamps and beat or bar positions
- reproducible normalization rules across songs with different loudness and density profiles

## Processing Steps

1. Load and validate the upstream harmonic, symbolic, energy, and section artifacts.
2. Normalize loudness, onset strength, spectral flux, spectral centroid trends, and stem-energy measures using documented per-song rules.
3. Compute derived features such as energy delta, density delta, silence gap length, vocal presence score, bass activation, and harmonic-tension proxies.
4. Align all feature layers to a shared time index while preserving bar and beat references when available.
5. Record rolling-window summaries for short, medium, and long windows so later stories can compare local versus contextual change.
6. Emit provenance metadata describing which upstream files and normalization rules produced the aligned feature layer.

## Output Artifacts

- `data/artifacts/<Song - Artist>/event_inference/features.json`
- optional documented helper index such as `data/artifacts/<Song - Artist>/event_inference/timeline_index.json`

## Acceptance Criteria

- All normalized features are exported in a reproducible, cross-song comparable format.
- Every feature layer can be queried on a shared timeline without manual timestamp adjustment.
- Bar and beat references are preserved when the upstream inputs provide them.
- Downstream stories can trace any feature value back to its source layer and normalization method.

## Validation

- Confirm all aligned windows stay within the canonical song duration.
- Confirm repeated runs on the same inputs produce identical normalized values and alignment indices.
- Confirm the derived features match the documented formulas and window sizes.

## Failure Cases

- Normalization silently depends on song-specific heuristics that are not recorded.
- Feature layers cannot be queried on a common timeline.
- Downstream event logic must reimplement alignment because the exported feature layer is incomplete.

## Out of Scope

- Final event labeling
- Human review workflow
- Fixture-aware lighting mapping
