# EPIC 1 - Story 1.2

## Beat and Tempo Detection

## Goal

Create the canonical timing grid used by all beat-aligned and bar-aligned downstream artifacts.

## Inputs

- Source song from `data/songs/<Song - Artist>.mp3`
- Optional stems from Story 1.1 when a stem-assisted timing workflow is needed

## Dependencies

- Essentia rhythm analysis
- Optional beat post-processing heuristics

## Processing Steps

1. Detect tempo in BPM.
2. Extract beat timestamps across the full song.
3. Infer bar groupings using the project time signature assumption.
4. Label downbeats and ordinary beats.
5. Store the canonical beat grid and bar grid under the artifact directory.
6. Reuse this grid as the only valid alignment target for later stories that snap events to beats or bars.

## Output Artifacts

- `data/artifacts/<Song - Artist>/essentia/beats.json`
- optional `data/artifacts/<Song - Artist>/essentia/bars.json` if stored separately
- timing metadata recorded into `info.json` and later merged artifacts

Generated beat artifacts must never use a `reference/` subfolder. `reference` is reserved for `/data/reference/` only.

## Output Schema

```json
{
  "schema_version": "1.0",
  "song_id": "Song - Artist",
  "tempo": 124.0,
  "time_signature": "4/4",
  "beats": [
    {"index": 1, "time": 0.512, "bar": 1, "beat_in_bar": 1, "type": "downbeat"},
    {"index": 2, "time": 0.996, "bar": 1, "beat_in_bar": 2, "type": "beat"}
  ],
  "bars": [
    {"bar": 1, "start_s": 0.512, "end_s": 2.450}
  ]
}
```

## Acceptance Criteria

- Beat timestamps are musically plausible across the full song.
- Downbeats are labeled consistently.
- Bars are 1-indexed and span the entire song without overlap.
- Downstream stories can snap events to this grid without inventing a second timing system.

## Validation

- Compare beat density and bar count against song duration and BPM.
- Spot-check downbeats at section starts.
- Compare inferred beat and bar outputs against validation-only reference beat files when they are available.
- Reference beat files are optional, read-only validation inputs and must never overwrite inferred beat or bar artifacts.

## Failure Cases

- Beat drift in long instrumental passages
- Downbeat misclassification
- Missing beats at the start or tail of the song

## Out of Scope

- Genre-specific meter changes
- Human annotation tooling
