# EPIC 5 - Story 5.2

## Energy and Musical Features to Lighting Mapping

## Goal

Translate the unified `music_feature_layers.json` artifact into normalized lighting behaviors before fixture-specific orchestration is applied.

## Inputs

- `music_feature_layers.json` from Story 5.1

## Dependencies

- Internal rule engine
- Deterministic normalization rules
- Smoothing and transition logic

## Processing Steps

1. Load `music_feature_layers.json` as the only required upstream analysis input.
2. Normalize musical and energy features into shared control dimensions.
3. Map section type, energy level, accent candidates, brightness trend, and symbolic motion into abstract lighting parameters.
4. Generate time-based lighting events that are fixture-agnostic.
5. Apply smoothing to avoid noisy or jittery behavior.

## Feature Mapping Contract

| Input feature | Abstract lighting dimension |
| --- | --- |
| loudness and energy level | intensity |
| spectral centroid and brightness trend | color temperature or hue bias |
| spectral flux and transient density | movement speed |
| accent windows and onset anchors | flash or strobe accents |
| note density and repetition | effect complexity |
| bass motion | pulse behavior |
| section label and trend | scene selection |

## Output Artifacts

- fixture-agnostic `lighting_events.json` or equivalent intermediate lighting event file

## Output Schema

```json
{
  "lighting_events": [
    {
      "time": 60.0,
      "duration": 4.0,
      "scene": "chorus_drive",
      "intensity": 0.84,
      "movement_speed": 0.72,
      "color_family": "cool",
      "accent_mode": "onset_sync"
    }
  ]
}
```

## Acceptance Criteria

- Lighting responses are synchronized with the analysis timeline.
- Higher-energy sections are measurably more intense than lower-energy sections.
- The output remains fixture-agnostic and reusable by Story 5.3.

## Validation

- Confirm Story 5.2 consumes only the unified Story 5.1 artifact, not the raw upstream layer files directly.
- Compare scene changes and intensity shifts against section and accent windows.
- Verify smoothing does not erase intended accents.

## Failure Cases

- Feature normalization collapses all sections into similar outputs
- Accent density causes chaotic event generation
- Story 5.2 bypasses `music_feature_layers.json` and recreates raw cross-layer joins

## Out of Scope

- Fixture addressing and channel mapping
- Final prose lighting score generation
