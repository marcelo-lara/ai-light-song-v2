# EPIC 1 - Story 1.3

## FFT Band Extraction

## Goal

Extract a compact low-to-high spectral intensity surface from the source song so the internal debugger can inspect broad frequency movement over time without introducing a new `data/output/` contract file.

## Inputs

- Source song from `data/songs/<Song - Artist>.mp3`
- Optional timing metadata from Story 1.2 for contextual inspection only

## Dependencies

- Essentia audio loading and FFT spectrum analysis
- NumPy for band aggregation and normalization
- Docker runtime with the analyzer dependency stack available

## Processing Steps

1. Load the source song as mono at the canonical analysis sample rate.
2. Step through the song at fixed 50 ms intervals.
3. Compute a magnitude spectrum for each frame.
4. Aggregate spectral magnitude into seven fixed musical bands ordered from low to high.
5. Normalize each band's level across the song to a stable `0.0` to `1.0` range.
6. Persist the artifact under the Essentia producer namespace.
7. Record the artifact path in `data/output/<Song - Artist>/info.json` for debugger discovery without adding a new file to `data/output/<Song - Artist>/`.

## Output Artifacts

- `data/artifacts/<Song - Artist>/essentia/fft_bands.json`

Generated FFT band artifacts must remain under `data/artifacts/<Song - Artist>/`. Do not add a new routine file under `data/output/<Song - Artist>/` for this story.

## Acceptance Criteria

- The artifact is generated directly from the source MP3.
- Exactly seven low-to-high bands are recorded in stable order.
- **Zero-Anchor:** Frame 0 must start at `0.0`. If the audio is silent, levels for early frames must be `0.0` but the frames must still be present.
- Frame timestamps advance in 50 ms increments.
- Levels are normalized into a stable `0.0` to `1.0` range for debugger rendering.
- The debugger can read the artifact directly from `data/artifacts/<Song - Artist>/essentia/fft_bands.json`.
- No new routine file is added under `data/output/<Song - Artist>/`.

## Validation

- Confirm the artifact file exists and is non-empty.
- Confirm `bands` contains exactly seven entries with stable ids and frequency edges.
- Confirm `metadata.interval_ms` is `50` and `metadata.hop_size` is `2205` at 44.1 kHz.
- Confirm every frame carries seven level values.
- Spot-check that band levels change over musically active passages instead of remaining flat for the entire song.

## Failure Cases

- Missing source song file
- Essentia import or FFT dependency failure
- Empty or unreadable audio input
- Artifact generated with fewer or more than seven bands
- Non-monotonic or non-50 ms frame timestamps

## Out of Scope

- Beat-aligned or bar-aligned FFT projections
- Event inference or lighting logic that consumes the FFT bands directly
- Alternate band presets or per-genre band layouts
- Adding a new stable file under `data/output/<Song - Artist>/`
