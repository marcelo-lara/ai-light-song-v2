# EPIC 5 - Story 5.9

## Optional ML Event Classifier and Explainability

## Goal

Train or tune a learned event classifier from benchmark data and preserve concise, interpretable explanations for its decisions.

## Inputs

- benchmark annotations from Story 5.7
- normalized event feature layer from Story 5.2
- rule candidates from Story 5.3
- reviewed machine or merged events from Story 5.6

## Dependencies

- a reproducible training dataset built from reviewed songs
- deterministic evaluation splits and metrics
- explanation output that exposes top contributing features or nearby evidence windows

## Processing Steps

1. Build a reproducible dataset from reviewed benchmark songs.
2. Train and evaluate one or more candidate approaches such as sequence labeling, window classification, or a hybrid rules-plus-ML pipeline.
3. Compare learned output against the rule baseline on key event types and timing accuracy.
4. Export classifier decisions with top contributing features, candidate labels, and concise explanation text.
5. Preserve a stable fallback path so low-confidence ML decisions can defer to rule or parent-label outputs.

## Output Artifacts

- `models/event_classifier/`
- `data/artifacts/<Song - Artist>/event_inference/events.ml.json`
- `data/artifacts/<Song - Artist>/event_inference/explanations.json`

## Acceptance Criteria

- A reproducible dataset exists for training and evaluation.
- The learned classifier can match or improve on the rules baseline for the targeted event types.
- Exported decisions include concise explanations rather than opaque scores only.

## Validation

- Compare learned output against the rules baseline and reviewed benchmark annotations.
- Confirm explanation artifacts identify the main evidence windows or contributing features for each accepted event.
- Confirm low-confidence cases can fall back cleanly without schema changes.

## Failure Cases

- Training data cannot be reproduced from reviewed artifacts.
- The classifier emits labels that are not documented in the canonical vocabulary.
- Decisions are exported as black boxes with no usable explanation fields.

## Out of Scope

- End-user review UI
- Fixture-specific lighting logic
- Freeform genre storytelling without documented labels
