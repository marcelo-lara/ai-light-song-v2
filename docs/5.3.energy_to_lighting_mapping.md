# EPIC 5 - Story 5.3

## Energy and Musical Features to Lighting Mapping

## Goal

Translate the unified `music_feature_layers.json` artifact into normalized lighting behaviors and deterministic cue anchors before fixture-specific orchestration is applied.

## Inputs

- `music_feature_layers.json` from Story 5.2

## Dependencies

- Internal rule engine
- Deterministic normalization rules
- Smoothing and transition logic

## Processing Steps

1. Load `music_feature_layers.json` as the only required upstream analysis input.
2. Normalize musical and energy features into shared control dimensions.
3. Map section type, energy level, accent candidates, brightness trend, symbolic motion, motif repetition, and harmonic pattern recurrence into abstract lighting parameters.
4. Generate exact time-based cue anchors from section, phrase, accent, and repeated-pattern windows.
5. Generate time-based lighting events that are fixture-agnostic.
6. Apply smoothing to avoid noisy or jittery behavior without destroying required anchor points.

## Feature Mapping Contract

| Input feature | Abstract lighting dimension |
| --- | --- |
| loudness and energy level | intensity |
| spectral centroid and brightness trend | color temperature or hue bias |
| spectral flux and transient density | movement speed |
| accent windows and onset anchors | flash or strobe accents |
| note density and repetition | effect complexity |
| phrase or motif recurrence | callback or controlled variation |
| bass motion | pulse behavior |
| harmonic pattern occurrence | scene recall or evolving callback |
| section label and trend | scene selection |

## Output Artifacts

- fixture-agnostic `lighting_events.json` or equivalent intermediate lighting event file

Current canonical path:

- `data/artifacts/<Song - Artist>/lighting_events.json`

The lighting-mapping layer should normalize downstream references using these field names:

- `lighting_events[].anchor_refs.section_id`
- `lighting_events[].anchor_refs.phrase_window_id`
- `lighting_events[].anchor_refs.motif_group_id`
- `lighting_events[].anchor_refs.pattern_id`
- `lighting_events[].anchor_refs.cue_anchor_ids[]`
- `cue_anchors[]`
- `cue_anchors[].id`
- `cue_anchors[].time_s`
- `cue_anchors[].anchor_type`

## Acceptance Criteria

- Lighting responses are synchronized with the analysis timeline.
- Higher-energy sections are measurably more intense than lower-energy sections.
- Repeated motifs and repeated harmonic patterns can drive repeatable callback logic with controlled variation.
- The output remains fixture-agnostic and reusable by Story 5.4.

## Validation

- Confirm Story 5.3 consumes only the unified Story 5.2 artifact, not the raw upstream layer files directly.
- Compare scene changes and intensity shifts against section and accent windows.
- Compare motif and pattern callbacks across repeated occurrences for consistency.
- Verify smoothing does not erase intended accents or exact cue anchors.

## Failure Cases

- Feature normalization collapses all sections into similar outputs
- Accent density causes chaotic event generation
- Repeated motifs or harmonic loops are ignored, causing structurally inconsistent cue behavior
- Story 5.3 bypasses `music_feature_layers.json` and recreates raw cross-layer joins

## Out of Scope

- Fixture addressing and channel mapping
- Final prose lighting score generation
