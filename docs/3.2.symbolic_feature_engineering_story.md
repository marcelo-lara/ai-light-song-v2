# EPIC 3 - Story 3.2

## Symbolic Feature Engineering

## Goal

Transform note events into reusable symbolic descriptors for musical interpretation and later lighting logic.

## Inputs

- Note events from Story 3.1
- Beat and bar grid from Story 1.2
- Section windows from Story 4.2

## Dependencies

- Rule-based feature derivation on top of aligned note events

## Processing Steps

1. Compute note density per beat and per bar.
2. Measure active-note counts, pitch range, and register centroid.
3. Derive melodic contour at phrase and section scales.
4. Derive bass motion descriptors from the bass-linked event subset.
5. Compute repetition and sustain metrics.
6. Produce global symbolic summaries and section-local summaries.

## Output Artifacts

- enriched `layer_b_symbolic.json`

## Output Schema

```json
{
  "symbolic_summary": {
    "texture": "layered",
    "melodic_contour": "rising",
    "bass_motion": "leaping",
    "repetition_score": 0.31,
    "sustain_ratio": 0.38
  },
  "density_per_bar": [
    {"bar": 1, "density": 0.42, "section_name": "Intro"}
  ]
}
```

## Acceptance Criteria

- Global summaries are understandable by a musician.
- Per-bar and per-section descriptors are internally consistent with the note events.
- Feature output is stable enough to support textual abstraction and lighting mapping.

## Validation

- Confirm pitch range and centroid match the overall register of the song.
- Confirm repetition metrics correlate with repeated phrase regions.
- Confirm section summaries are recomputed from local data, not copied from global summaries.

## Failure Cases

- Feature scaling collapses all songs to the same generic summary
- Phrase grouping is too unstable to support contour descriptions
- Empty sections cause invalid summary objects

## Out of Scope

- Genre-specific symbolic ontology
- Advanced motif discovery beyond the documented descriptor set
