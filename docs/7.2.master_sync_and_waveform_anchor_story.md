# EPIC 7 - Story 7.2

## Master Sync and Waveform Anchor

## Goal

Establish the internal debugger playback surface and the shared timeline source of truth for artifact inspection.

This story is for the internal artifact debugger under `/ui/`. It is not a production consumer UI story.

## Inputs

- `data/songs/<Song - Artist>.mp3`
- `data/output/<Song - Artist>/info.json`
- `data/artifacts/<Song - Artist>/essentia/beats.json`

## Processing Steps

1. Load song metadata and timing context for a selected song directory.
2. Initialize the debugger playback shell from the `/ui` application.
3. Treat audio playback time as the single source of truth for the shared cursor.
4. Expose playhead updates to downstream debugger lanes and overlays.

## Output

- browser-only playback state inside the debugger

No generated files are written to `data/artifacts/` or `data/output/`.

## Acceptance Criteria

- The debugger can load a song and display core metadata.
- The playback surface runs in the separate Compose `ui` service.
- Playback state is read-only against generated data.
- No debugger-generated files are written into generated-data folders.
