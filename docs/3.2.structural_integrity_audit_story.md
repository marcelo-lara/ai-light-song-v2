# EPIC 3 - Story 3.2
## Structural Integrity and Alignment Audit

## Goal
Perform a high-precision audit on inferred structural boundaries to ensure they align with measurable audio transients and respect human-provided reference data without losing the benefits of machine inference. This story provides the section-level `confidence_score` consumed by downstream abstractions and control logic.

## Intent

This story owns section-boundary audit only. It does not define event-level alignment scoring or downstream lighting policy.

## Inputs
- **Inferred Sections** (Story 3.1): Boundaries detected via Neural SSM.
- **Spectral Transients** (Story 1.3): Raw `fft_bands.json` data containing sub-band energy spikes.
- **Reference Sections** (Optional): Read-only human-annotated `.json` files (e.g., from `benchmark_annotations/` or a User Hint file).

## Processing Steps
1.  **Transient Correlation (Leading-Edge Snapping)**:
    - Scan the area around each inferred boundary (±100ms).
    - Identify the nearest leading-edge spectral transient (spike) in the `fft_bands.json`.
    - If a transient exists within 50ms, flag the boundary as "Locked." 
    - If the nearest transient is > 50ms away, flag a **Validation Warning: Transient Mismatch**.
2.  **Reference Verification (Non-Destructive)**:
    - Load the optional `reference_sections.json`.
    - Calculate the **Jaccard Similarity** (Intersection over Union) between the Inferred and Reference section sets.
    - Compute a "Structure Alignment Score" (0.0–1.0). 
    - **CRITICAL**: Do NOT overwrite the inferred boundaries with reference data. The reference is used only for scoring and validation.
3.  **Confidence Metadata Injection**:
    - For every section, compute a final `confidence_score` based on:
        - Distance to nearest FFT transient.
        - Clarity of the Novelty Peak in Story 3.1.
        - Agreement with Reference Sections (if available).

## Output Artifacts
- `data/artifacts/<Song>/section_segmentation/audit_report.json`: Detailed alignment report and mismatch logs.
- Updated `sections.json`: Now includes `confidence_score` and `is_locked_to_transient` boolean per segment.

## Acceptance Criteria
- **Transient Locking**: Boundaries that are naturally "locked" to a kick or snare hit are identified as high-confidence.
- **Audit Integrity**: The audit process must never modify the timing of the inferred boundaries; it only provides metadata about their accuracy.
- **Reference Awareness**: If a human reference file is provided, the system must report exactly how much the AI "disagreed" with it.

## Validation
- Verify that a song with clear, rhythmic transients (e.g., EDM) achieves > 90% "Locked" status.
- Confirm that downstream abstractions such as Story 6.3 can consume `confidence_score` without guessing its meaning.
