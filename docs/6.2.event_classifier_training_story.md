# EPIC 6 - Story 6.2
## 1D-CNN Training and Dataset Generation

## Goal
Create a repeatable training pipeline to produce the weights for the 1D-CNN Event Classifier. This story converts reference `human_hints.json` windows and the 100ms contextual feature grid into a supervised multi-label dataset the neural network can learn from.

Training is conditional: the training routine should run only when `models/event_classifier/1d_cnn_v1.pth` does not already exist. If the weights file is already present, the pipeline should treat it as the current promoted artifact and skip retraining unless an explicit retraining workflow is requested.

## Inputs
- **Feature Streams**: `data/artifacts/<Song>/event_inference/contextual_features.json` (the 100ms "X" data produced by Story 5.2).
- **Human Truth**: `data/reference/<Song>/human/human_hints.json` (The "Y" labels).
- **Model Spec**: 1D-CNN Architecture defined in Story 6.1.

## Processing Steps
1. **Label Vectorization**:
    - Map `human_hints.json` event types (e.g., "Drop", "Build") to indices in a multi-hot vector based on the Story 5.1 Event Vocabulary.
    - For every 100ms feature frame, assign a 1 or 0 for each event class based on whether the timestamp falls within a human-annotated window.
2. **Windowed Dataset Creation**:
    - Implement a sliding window (256 steps @ 100ms resolution).
    - Sample windows containing events more frequently during training to handle class imbalance (since music is mostly "no event").
3. **Training Execution**:
    - Check whether `models/event_classifier/1d_cnn_v1.pth` already exists before starting training.
    - Skip training when that promoted weights file already exists, leaving the current artifact in place.
    - Split songs into Training (80%) and Validation (20%) sets.
    - Train using Binary Cross-Entropy (BCE) Loss with the Adam optimizer.
4. **Weight Export**:
    - Save the best-performing model (based on Validation F1-score, with precision gate enforcement) to `models/event_classifier/1d_cnn_v1.pth`.

## Output Artifacts
- `models/event_classifier/1d_cnn_v1.pth`: The serialized weights.
- `models/event_classifier/metadata.json`: Records which songs were used for training and the final accuracy metrics.

## Acceptance Criteria
- **Training Success**: The loss curve must show convergence over at least 20 epochs.
- **Validation**: The model must achieve a precision of >70% on the validation set before being promoted to `v1`.
- **Cinderella Test**: After training on a dataset including "Cinderella - Ella Lee" (or similar tracks), the model must detect the major energy shifts defined in its `human_hints.json`.
- **Skip Behavior**: If `models/event_classifier/1d_cnn_v1.pth` already exists, the default training path must skip retraining and preserve the existing promoted weights.

## Validation
- Run `python -m scripts.train_event_classifier --songs-root /data/reference --artifacts-root /data/artifacts`.
- When the weights file is absent, verify the `.pth` file is created and contains non-zero weights.
- When the weights file already exists, verify the default training path skips retraining rather than overwriting the promoted artifact.

## Implementation Notes

- The canonical training entrypoint is `python -m scripts.train_event_classifier`.
- The trainer consumes `contextual_features.json`, derives multi-hot labels from `human_hints.json`, and writes both weights and metrics under `models/event_classifier/`.
- Promotion to `1d_cnn_v1.pth` is gated on the validation precision threshold. Failed promotion still writes `metadata.json` so the result can be inspected without pretending the model passed Story 6.2 acceptance.
- The default Story 6.2 contract is artifact-aware: if `models/event_classifier/1d_cnn_v1.pth` already exists, training should not rerun unless an explicit retraining workflow is invoked.
