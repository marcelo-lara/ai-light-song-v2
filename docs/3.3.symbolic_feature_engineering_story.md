# EPIC 3 - Story 3.3

## Symbolic Feature Engineering

## Goal

Transform note events into reusable symbolic descriptors for musical interpretation and later lighting logic.

For downstream lighting use, this includes motif-level and repeated-phrase summaries that can explain why repeated musical material should trigger repeated or intentionally varied visual behavior.

## Inputs

- Note events from Story 3.1
- Symbolic validation summary from Story 3.1 when source promotion decisions are available
- Beat and bar grid from Story 1.2
- Section windows from Story 4.2

## Dependencies

- Rule-based feature derivation on top of aligned note events

## Processing Steps

1. Compute note density per beat and per bar.
2. Measure active-note counts, pitch range, and register centroid.
3. Derive melodic contour at phrase and section scales.
4. Derive bass motion descriptors from the bass-linked event subset.
5. Compute repetition and sustain metrics.
6. Derive motif-level or repeated-phrase summaries within the documented symbolic descriptor set.
7. Produce global symbolic summaries and section-local summaries.

## Output Artifacts

- enriched `layer_b_symbolic.json`

For downstream lighting use, the following field names are canonical when the data is available:

- `symbolic_summary.texture`
- `symbolic_summary.melodic_contour`
- `symbolic_summary.bass_motion`
- `symbolic_summary.repetition_score`
- `symbolic_summary.sustain_ratio`
- `density_per_beat[]`
- `motif_summary.dominant_motif_id`
- `motif_summary.motif_groups[]`
- `motif_summary.motif_groups[].id`
- `motif_summary.motif_groups[].occurrence_refs[]`
- `motif_summary.repeated_phrase_groups[]`
- `motif_summary.repeated_phrase_groups[].id`
- `motif_summary.repeated_phrase_groups[].phrase_window_ids[]`

## Acceptance Criteria

- Global summaries are understandable by a musician.
- Per-bar and per-section descriptors are internally consistent with the note events.
- Per-beat and per-bar density rows are derived from the canonical timing grid.
- Repetition summaries identify repeated phrase behavior clearly enough for later lighting callback rules.
- Feature output is stable enough to support textual abstraction and lighting mapping.

## Validation

- Confirm pitch range and centroid match the overall register of the song.
- Confirm repetition metrics correlate with repeated phrase regions.
- Confirm motif or repeated-phrase summaries remain consistent across repeated note-pattern regions.
- Confirm section summaries are recomputed from local data, not copied from global summaries.

## Failure Cases

- Feature scaling collapses all songs to the same generic summary
- Phrase grouping is too unstable to support contour descriptions
- Repetition summaries are too vague to distinguish callback-worthy material from one-off phrases
- Empty sections cause invalid summary objects

## Out of Scope

- Genre-specific symbolic ontology
- Advanced motif discovery beyond the documented descriptor set
