# EPIC 5 - Story 5.3

## Unified Music Feature Layer Assembly

## Goal

Assemble the upstream harmonic, symbolic, energy, and pattern layers into a single cross-layer artifact that downstream lighting stories can consume without reimplementing joins across multiple files.

In this assembly, symbolic motif-level repetition remains sourced from `layer_b_symbolic.json`, while Layer D contributes chord-progression repetition summaries.

The unified artifact must expose deterministic timing anchors strongly enough that Story 5.5 can generate a `lighting_score.md` with exact cue times instead of approximate prose.

## Inputs

- `layer_a_harmonic.json`
- `layer_b_symbolic.json`
- `layer_c_energy.json`
- `layer_d_patterns.json`
- canonical beat and bar grid from Story 1.2
- section windows from Story 4.2
- core metadata from `data/output/<Song - Artist>/info.json`

## Dependencies

- Stable upstream layer schemas
- Deterministic alignment logic across beats, bars, sections, and accent windows

## Processing Steps

1. Load and validate all upstream layer files.
2. Load shared timing data and confirm all layer times fall inside the canonical song duration.
3. Build a unified timeline object containing beats, bars, sections, phrase anchors, and accent windows.
4. Embed or reference the relevant harmonic, symbolic, energy, and pattern summaries.
5. Preserve the ownership boundary between symbolic motif summaries and harmonic progression-pattern summaries instead of collapsing them into one ambiguous field family.
6. Expose repeated-pattern occurrences, motif references, and timing anchors in a form that downstream lighting logic can use without re-deriving them.
7. Produce section cards and generation notes that make downstream lighting logic easier to implement.
8. Record `generated_from` metadata for all upstream sources.

## Output Artifacts

- `data/artifacts/<Song - Artist>/music_feature_layers.json`
- optional documented helper outputs only when they are explicitly referenced by later stories

The unified artifact should normalize cross-layer references using these canonical field names:

- `timeline.phrases[]`
- `timeline.phrases[].id`
- `timeline.phrases[].phrase_group_id`
- `lighting_context.cue_anchors[]`
- `lighting_context.cue_anchors[].id`
- `lighting_context.cue_anchors[].time_s`
- `lighting_context.cue_anchors[].anchor_type`
- `lighting_context.pattern_callbacks[]`
- `lighting_context.pattern_callbacks[].pattern_id`
- `lighting_context.pattern_callbacks[].callback_action`
- `lighting_context.motif_callbacks[]`
- `lighting_context.motif_callbacks[].motif_group_id`
- `lighting_context.motif_callbacks[].callback_action`

## Acceptance Criteria

- The file can act as the single required input for EPIC 5.4 and EPIC 5.5.
- All upstream dependencies are explicit in `generated_from`.
- Beats, bars, sections, phrase anchors, and accent windows are internally consistent.
- Pattern occurrences and motif references are exposed with enough timing detail to support exact downstream cue placement.
- Cross-layer timing and callback IDs are stable enough for Story 5.4 and Story 5.5 to reference them directly.
- The artifact does not invent new analysis data that is absent from upstream layers.

## Validation

- Confirm all referenced upstream files exist.
- Confirm section windows, phrase anchors, accent windows, and cue-anchor IDs match the source layer data.
- Confirm representative summary values in the unified artifact match their upstream source files.

## Failure Cases

- Layer joins silently drop a source layer
- Timeline arrays use a different indexing model than the canonical beat grid
- Phrase or pattern anchors are summarized without preserving exact usable times
- The unified artifact duplicates contradictory values from multiple sources without resolving ownership

## Out of Scope

- Creative lighting decisions
- DMX fixture control
