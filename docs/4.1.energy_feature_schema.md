# EPIC 4 - Story 4.1

## Energy Feature Schema

## Goal

Define the normalized low-level energy schema used by the energy layer and all later energy-derived stories.

## Inputs

- Source song audio or selected stems
- Canonical beat grid from Story 1.2 when beat-aligned summaries are generated

## Dependencies

- Essentia feature extraction

## Core Feature Frame Schema

```json
{
  "time": 1.230,
  "frame_index": 123,
  "loudness": 0.82,
  "spectral_centroid": 2450.5,
  "spectral_flux": 0.34,
  "onset_strength": 0.67
}
```

## Collection Schema

```json
{
  "features": [
    {
      "time": 0.000,
      "frame_index": 0,
      "loudness": 0.10,
      "spectral_centroid": 1200.0,
      "spectral_flux": 0.05,
      "onset_strength": 0.02
    }
  ],
  "metadata": {
    "frame_rate": 100,
    "total_frames": 18000,
    "duration": 180.0
  }
}
```

## Beat-Aligned Variant

```json
{
  "beat": 12,
  "time": 24.0,
  "loudness_avg": 0.75,
  "centroid_avg": 2100.0,
  "flux_avg": 0.40,
  "onset_density": 3.2
}
```

## Field Definitions

| Field | Type | Meaning |
| --- | --- | --- |
| `time` | float | Timestamp in seconds |
| `frame_index` | int | Analysis frame index |
| `loudness` | float | Normalized perceived loudness |
| `spectral_centroid` | float | Brightness proxy in Hz or normalized equivalent |
| `spectral_flux` | float | Frame-to-frame spectral change |
| `onset_strength` | float | Transient strength |

## Rules

- Preserve raw or near-raw low-level features before higher-level aggregation.
- Prefer storing both frame-level and beat-level views.
- Keep normalization consistent across songs.
- Do not treat this layer as semantic musical meaning; it represents physical energy and motion evidence.

## Downstream Consumers

- Story 4.2 section segmentation
- Story 4.3 derived energy features
- Story 5.1 unified music feature layer assembly

## Validation

- Confirm the frame sequence covers the analyzed song duration.
- Confirm beat-level aggregations align with the canonical beat grid.
- Confirm normalized ranges are stable enough for downstream rules.
