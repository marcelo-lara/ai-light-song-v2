# EPIC 2 - Story 2.3

## Key and Tonal Center Detection

## Goal

Infer the global key and optionally section-level tonal centers from harmonic evidence.

## Inputs

- HPCP features from Story 2.1
- Chord events from Story 2.2
- Section windows from Story 4.2 when local key estimation is enabled

## Dependencies

- Essentia key estimation or equivalent tonal-profile approach

## Processing Steps

1. Estimate the global key from the full-song harmonic representation.
2. Optionally estimate local tonal centers inside section windows.
3. Combine chord evidence and pitch-class prominence into a final key label and confidence score.
4. Persist the result into the harmonic layer and summary metadata.

## Output Artifacts

- key metadata within `layer_a_harmonic.json`
- mirrored summary fields in `info.json` and merged artifacts

## Output Schema

```json
{
  "global_key": {
    "label": "D major",
    "confidence": 0.77,
    "source": "chords+hpcp"
  }
}
```

## Acceptance Criteria

- Global key matches musician expectation for test songs when the material is tonally stable.
- Confidence decreases when tonal evidence is mixed or ambiguous.
- Local key estimation, when enabled, does not overwrite the global key contract.

## Validation

- Compare inferred key outputs against validation-only reference metadata when it is available.
- Reference metadata is optional, read-only validation input and must never overwrite inferred global or section-level key outputs.
- Confirm that dominant pitch classes and chord centers support the chosen label.

## Failure Cases

- Ambiguous modal material produces false certainty
- Section windows are too short for stable local key detection

## Out of Scope

- Full roman numeral functional analysis
- Modulation graph modeling across the whole song
