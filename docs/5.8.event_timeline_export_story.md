# EPIC 5 - Story 5.8

## LLM-Friendly Event Timeline Export

## Goal

Export a compact, lighting-relevant event timeline that summarizes machine and human-reviewed musical events without embedding raw feature arrays.

## Inputs

- merged or machine-only event outputs from Story 5.6
- canonical event vocabulary and schema from Story 5.1
- section references and supporting evidence summaries from upstream artifacts

## Dependencies

- a compact export shape suitable for downstream prompting
- stable merge rules when human overrides exist
- optional high-level lighting hints that remain editable and non-binding

## Processing Steps

1. Merge machine output with human overrides when they exist, using documented precedence rules.
2. Export a condensed event timeline in JSON and markdown with time range, event type, confidence, intensity, short notes, and evidence summary.
3. Attach optional high-level lighting hints for event types such as `drop_explode`, `anthem_call`, `groove_loop`, or `atmospheric_plateau`.
4. Keep raw arrays and detailed supporting evidence in separate files and reference them instead of embedding them.
5. Preserve whether each event is machine-only, human-confirmed, or human-edited.

## Output Artifacts

- `data/output/<Song - Artist>/song_event_timeline.json`
- `data/output/<Song - Artist>/song_event_timeline.md`

## Acceptance Criteria

- The exported timeline is compact enough for LLM or planning use.
- Each event includes enough context for show-planning decisions without requiring raw-array inspection.
- Lighting hints remain high-level and editable rather than fixture-specific mandates.
- The export preserves review provenance for each event.

## Validation

- Confirm the JSON and markdown outputs contain the same event set and timing.
- Confirm timeline entries reference supporting evidence rather than duplicating large feature payloads.
- Confirm human-authored show notes can override generic lighting hints when present.

## Failure Cases

- The export embeds raw feature arrays and becomes too large for downstream prompting.
- Lighting hints become fixture-specific instructions.
- Provenance is lost during machine and human merge.

## Out of Scope

- Fixture-aware orchestration
- ML training and evaluation
- Real-time control transport
