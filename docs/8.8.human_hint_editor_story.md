# EPIC 8 - Story 8.8

## Human Hint Editor

## Goal

Add a right sidebar for creating, editing, and deleting reference human hints inside the helper UI.

The sidebar is initially hidden. It opens when the user clicks a human hint on the timeline or explicitly starts a new hint from the helper UI.
When open for an existing hint, the sidebar displays only the currently selected human hint.

## Inputs

- `data/reference/<Song - Artist>/human/human_hints.json`
- the helper UI human-hints lane derived from that file
- the current timeline cursor shared by playback and timeline scrubbing

## Interaction Model

- The right sidebar is hidden on initial page load.
- Clicking a human hint opens the right sidebar with that hint selected.
- Right-clicking on the "Human Hints" lane background opens the sidebar and begins drafting a new human hint at the clicked time.
- Human hint clicks do not open the standard selection popup; human hints use the sidebar as their only detail surface.
- Opening or keeping the human hint sidebar open must not block later timeline scrubbing or cursor repositioning, including when playback is paused.
- Existing human hints are selected only from the timeline.
- The sidebar displays only the currently selected human hint instead of a full human-hint list.
- The editor exposes `title`, `start_time`, `end_time`, `summary`, and `lighting_hint`. The `id` is generated under the hood sequentially (e.g. `hint-001`, `hint-002`) and kept hidden from the UI.
- The `start_time` and `end_time` fields each expose a `Set` action positioned to the left of the input that copies the current timeline cursor time into that field.
- The sidebar does not close on lost focus, overlay dismissal, or unrelated pointer movement.
- The only close paths are `Save` and `Cancel`.
- `Delete` is a draft action that removes the active hint from local state but leaves the sidebar open (showing a Save/Cancel prompt) until the user explicitly saves or cancels.
- The timeline instantly reflects all draft edits, additions, and deletions as a live preview before they are saved.
- Human-hint editor text should use a smaller presentation than the standard detail surfaces so the form fits comfortably in the sidebar.
- The sidebar layout stays compact: avoid nested cards and avoid paddings greater than `0.5em` inside the editor surface.
- `Cancel` closes the sidebar and discards unsaved local edits, reverting the timeline to the saved state.
- `Save` persists the current editor state to disk and closes the sidebar.

## Persistence Rules

- `Save` updates only `data/reference/<Song - Artist>/human/human_hints.json`.
- `Cancel` does not update the file.
- No writes are permitted to `data/artifacts/`.
- No writes are permitted to `data/output/`.
- The helper UI does not auto-save human hint edits.

## Acceptance Criteria

- The human hint editor is hidden by default.
- Clicking a human hint opens the right sidebar with the selected hint loaded.
- Right-clicking on the empty space of the "Human Hints" lane opens the sidebar and begins a new draft at the target time.
- Clicking a human hint does not open the standard selection popup.
- The sidebar shows only the selected human hint for existing-hint editing.
- Existing human hints are not re-selected from controls inside the sidebar.
- The sidebar remains open when focus moves away from the editor.
- `Save` and `Cancel` are the only supported ways to close the sidebar.
- `Delete` is a draft action; it stages the deletion for review but does not auto-save or auto-close.
- The timeline renders a live preview of all unsaved draft changes.
- The user can create, edit, and delete human hints in local editor state before saving.
- The `Set` actions copy the current timeline cursor time into the selected time field.
- The editor typography and spacing are compact enough to fit comfortably inside the sidebar without nested cards or padding greater than `0.5em`.
- `Cancel` closes the sidebar without updating the file.
- `Save` updates only `data/reference/<Song - Artist>/human/human_hints.json`.

## Validation

- Confirm the helper UI does not write to `data/artifacts/` or `data/output/`.
- Confirm the human hints lane renders draft state as a live preview instantly during creation, modification, and deletion.
- Confirm the human hints lane reflects the saved reference file after `Save` succeeds.
- Confirm `Cancel` restores the last saved reference state and dismisses the drafts.
- Confirm `Delete` does not automatically update the file or close the sidebar.
- Confirm the editor remains explicit-save only.
- Confirm right-clicking on the empty lane creates a new human hint at the right-clicked time and opens the editor.
- Confirm existing-hint editing is driven by timeline selection and not by a sidebar hint picker.
- Confirm human hints suppress the standard popup and use only the sidebar for detail/edit interaction.
- Confirm the timeline cursor can still be repositioned while playback is paused, even after opening the human hint sidebar from an existing hint.
- Confirm the sidebar stays open across focus changes until `Save` or `Cancel` is invoked.
- Confirm the sidebar uses the compact text and spacing rules documented in this story.

## Failure Cases

- Missing `human_hints.json` should leave the lane empty until the user creates and saves hints.
- Invalid JSON in the reference file should fail explicitly instead of silently replacing the file with generated defaults.
- Save failures should surface an explicit error in the helper UI.
- Invalid time ranges must not be persisted.

## Out of Scope

- Writes to generated artifacts or stable outputs
- auto-save behavior
- broader review-state persistence outside `human_hints.json`
- editing any reference material other than human hints
