# EPIC 8 - Story 8.9

## Identifier and ML Event Lanes

## Goal

Add read-only debugger inspection lanes that expose rule-driven identifier hints, Story 5.3 ML event predictions, and penalty or alignment outcomes on the shared timeline.

## Inputs

- `data/artifacts/<Song - Artist>/energy_summary/hints.json`
- `data/artifacts/<Song - Artist>/event_inference/events.ml.json`
- `data/artifacts/<Song - Artist>/event_inference/penalty_timeline.json` when available
- shared playback, zoom, and scroll state from Stories 8.2 and 8.3

## Processing Steps

1. Load `energy_summary/hints.json` as a sparse region lane using each event's `start_s`, `end_s`, `identifier`, `confidence`, `notes`, and `evidence` fields when present.
2. Load `event_inference/events.ml.json` as a separate sparse region lane using the ML event windows, labels, confidences, and any explainability metadata exported by Story 5.3.
3. Load `penalty_timeline.json` as a read-only heatmap or severity lane when the artifact is present so reviewers can inspect where physical evidence penalized or rewarded model predictions.
4. Keep all three inspection surfaces visually distinct from the existing machine-events lane so rule identifier hints, rule or machine events, ML predictions, and penalty outcomes can be compared without ambiguity.
5. Reuse the shared selection and navigation behavior from the debugger without introducing any writes to generated artifacts.

## Acceptance Criteria

- The debugger renders a dedicated `Identifier Hints` lane sourced only from `data/artifacts/<Song - Artist>/energy_summary/hints.json`.
- The debugger renders a dedicated `ML Events` lane sourced only from `data/artifacts/<Song - Artist>/event_inference/events.ml.json`.
- When `penalty_timeline.json` exists, the debugger renders a dedicated penalty heatmap or severity lane sourced only from that file.
- Both lanes align to the shared timeline clock and zoom model used by the rest of the debugger.
- If either artifact is missing, the corresponding lane fails explicitly or renders an empty state without substituting another source.
- If `penalty_timeline.json` is absent, the penalty lane renders an explicit empty state rather than synthesizing data from another artifact.
- Selecting regions in either lane affects only local debugger playback and inspection state.
- Neither lane writes to `data/artifacts/` or `data/output/`.

## Validation

- Confirm `Identifier Hints` windows match the timings and labels in `energy_summary/hints.json`.
- Confirm `ML Events` windows match the timings and labels in `event_inference/events.ml.json`.
- Confirm penalty heatmap or severity values match `penalty_timeline.json` when that file is present.
- Confirm all inspection lanes remain readable alongside the existing machine-events lane at the supported zoom levels.
- Confirm missing or invalid JSON in any source artifact surfaces an explicit debugger error instead of silently swapping to another file.
