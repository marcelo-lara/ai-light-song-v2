# EPIC 8 - Story 8.9

## Identifier and ML Event Lanes

## Goal

Add two read-only debugger lanes that expose the rule-driven identifier hints and the Story 6.1 ML event predictions on the shared timeline.

## Inputs

- `data/artifacts/<Song - Artist>/energy_summary/hints.json`
- `data/artifacts/<Song - Artist>/event_inference/events.ml.json`
- shared playback, zoom, and scroll state from Stories 8.2 and 8.3

## Processing Steps

1. Load `energy_summary/hints.json` as a sparse region lane using each event's `start_s`, `end_s`, `identifier`, `confidence`, `notes`, and `evidence` fields when present.
2. Load `event_inference/events.ml.json` as a separate sparse region lane using the ML event windows, labels, confidences, and any explainability metadata exported by Story 6.1.
3. Keep both lanes visually distinct from the existing machine-events lane so rule identifier hints, rule/machine events, and ML predictions can be compared without ambiguity.
4. Reuse the shared selection and navigation behavior from the debugger without introducing any writes to generated artifacts.

## Acceptance Criteria

- The debugger renders a dedicated `Identifier Hints` lane sourced only from `data/artifacts/<Song - Artist>/energy_summary/hints.json`.
- The debugger renders a dedicated `ML Events` lane sourced only from `data/artifacts/<Song - Artist>/event_inference/events.ml.json`.
- Both lanes align to the shared timeline clock and zoom model used by the rest of the debugger.
- If either artifact is missing, the corresponding lane fails explicitly or renders an empty state without substituting another source.
- Selecting regions in either lane affects only local debugger playback and inspection state.
- Neither lane writes to `data/artifacts/` or `data/output/`.

## Validation

- Confirm `Identifier Hints` windows match the timings and labels in `energy_summary/hints.json`.
- Confirm `ML Events` windows match the timings and labels in `event_inference/events.ml.json`.
- Confirm both lanes remain readable alongside the existing machine-events lane at the supported zoom levels.
- Confirm missing or invalid JSON in either source artifact surfaces an explicit debugger error instead of silently swapping to another file.
